Linux Essentials: Building a Secure Foundation from Scratch

2025-04-12 · 6 min read · 3rd3ye

Whether you're setting up a VPS, a home lab, or a CTF machine, starting with a hardened baseline saves significant pain later. This post walks through the essential steps I take on every fresh Linux install.

User and Privilege Management

Never run as root for daily tasks. The first thing to do on a new system:

# Create a non-root user
adduser analyst
usermod -aG sudo analyst

# Lock the root account
passwd -l root

Review which users have sudo access:

grep -Po '^sudo.+:\K.*$' /etc/group | tr ',' '\n'

SSH Hardening

Default SSH configurations are dangerously permissive. Edit /etc/ssh/sshd_config:

PermitRootLogin no
PasswordAuthentication no
PubkeyAuthentication yes
MaxAuthTries 3
AllowUsers analyst
Protocol 2

Generate a strong key pair on your local machine and copy the public key:

ssh-keygen -t ed25519 -C "lab-key"
ssh-copy-id -i ~/.ssh/id_ed25519.pub analyst@target

Firewall with UFW

UFW (Uncomplicated Firewall) wraps iptables in a sane interface:

# Default deny inbound, allow outbound
ufw default deny incoming
ufw default allow outgoing

# Allow only what you need
ufw allow 22/tcp comment 'SSH'
ufw enable

# Check status
ufw status verbose

Audit Logging with auditd

auditd gives you kernel-level visibility into system calls:

apt install auditd audispd-plugins

# Watch for privilege escalation attempts
auditctl -w /etc/sudoers -p wa -k sudoers_change
auditctl -w /etc/passwd -p wa -k passwd_change

# Watch for suspicious executions
auditctl -a always,exit -F arch=b64 -S execve -k exec_log

systemctl enable auditd
systemctl start auditd

File Integrity Monitoring

AIDE (Advanced Intrusion Detection Environment) creates a baseline of your filesystem and alerts on changes:

apt install aide
aideinit
cp /var/lib/aide/aide.db.new /var/lib/aide/aide.db

# Run a check
aide --check

What's Next

These steps form the foundation. From here, the next layer involves:

  • CIS Benchmark compliance
  • Kernel hardening via sysctl
  • AppArmor or SELinux profiles
  • Centralised log shipping (rsyslog → SIEM)

A hardened system isn't a one-time task — it's a practice.
